## 架构决策

- **后端路由策略**：本地代理优先（API Key 不暴露到客户端），DashScope 直连作为降级方案
- **API Key 管理**：三重后备 — dart-define（生产） → 环境变量（代理） → dev 常量（POC 本地）
- **异步 API 暂不可用**：DashScope 异步 endpoint 需要公开可访问的图片 URL，当前 POC 阶段无图片托管，通过 data: URL 方案尝试绕过（待验证），优先使用同步 base64 模式
- **Phase B 迁移计划**：接入 Supabase Storage 后，图片自动获得公开 URL，届时可切换到异步 API，支持上衣+下装单次提交

- [x] **DashScope OSS 上传 + 异步试衣流水线端到端验证通过**：
  - DashScope 所有试衣 API 仅接受公网 URL，不支持 base64
  - 利用 DashScope 内置临时文件上传 API（48h 有效，免费）
  - 流水线：本地图片 → 代理 → DashScope OSS → aitryon-plus 异步 → 轮询 → 返回
  - 实测：demo 人物+上衣，90.55s 生成，103KB JPEG ✅
- [x] **Web 版 POC 页面兼容性修复**：移除所有 `dart:io` 依赖，全用 `Uint8List`
- [x] **代理服务器 v0.3.0**：`requests` 库处理 OSS multipart 上传（aiohttp 有兼容问题）

## 关键发现

- **DashScope 试衣 API 不接受 base64 图片**，必须用公网 HTTP URL
- **DashScope 提供免费临时文件上传**（`/api/v1/uploads`），48 小时有效，上传后得 `oss://` URL
- **调用时必须加 `X-DashScope-OssResourceResolve: enable`** 头让 API 解析 `oss://` URL
- **aiohttp FormData 与阿里 OSS 的 multipart 解析不兼容**，改用 `requests` 库解决
- 当前架构：Flutter → 代理服务器 → DashScope OSS 上传 → 异步 API → 轮询 → 返回
