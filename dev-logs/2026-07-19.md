## 部署架构变更

### 原方案（Render，已放弃）
- 单 Docker 容器：Flutter Web + Python AI Server
- 问题：免费计划内存不足，Flutter Web 编译反复失败

### 新方案（Vercel + Render）
- **前端**：Flutter Web → Vercel（自动部署，全球 CDN）
- **后端**：Python AI Server → 暂留 Render（只提供 API）

---

## Bug 修复

### 1. pubspec.yaml 编码错误
- **问题**：`description` 字段中文字符被截断，导致 UTF-8 解码失败
- **修复**：补全为 `"穿搭辅助App - 虚拟试衣、AI搭配推荐、衣柜管理"`

### 2. api_client.dart 乱码 + 构造函数语法
- **问题**：文件中有大量 `ef bfbd 3f`（UTF-8 替换字符），dart2js 解析失败
- **修复**：重写整个文件，清除乱码注释，构造函数改用简单默认值

### 3. GitHub Action Flutter SDK 版本
- **问题**：pubspec.yaml 要求 Dart SDK `^3.10.7`，stable channel 只有 3.5.4
- **修复**：改用 `channel: 'master'` 获取最新 Flutter

### 4. Vercel CLI 版本过旧
- **问题**：`amondnet/vercel-action@v25` 内置 Vercel CLI 25.1.0，API 要求 47.2.2+
- **修复**：弃用第三方 action，改用 `npm i -g vercel@latest` 直接部署
