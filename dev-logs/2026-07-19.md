## 技术实现

### 多阶段 Docker 构建

**Stage 1：Flutter Web 编译**
- 使用 `ghcr.io/cirruslabs/flutter:stable` 镜像
- 执行 `flutter build web --release`
- 输出静态文件到 `/app/build/web/`

**Stage 2：Python AI Server**
- 使用 `python:3.11-slim` 镜像
- 安装 rembg 依赖（libgl1-mesa-glx, libglib2.0-0）
- 复制 Flutter Web 静态文件到 `/app/static/`
- 启动时通过 `--static-dir` 参数加载静态文件

### AI 服务器升级（v0.7.0）

新增功能：
- `--static-dir` 命令行参数：指定 Flutter Web 静态文件目录
- `setup_static_files()` 函数：配置静态文件服务
- SPA 路由支持：非 API 路径返回 `index.html`
- 自动排除 `/api/*` 路径

### Flutter API 配置优化

- `AI_PROXY_URL` 环境变量为空时自动使用相对路径
- 生产环境前后端同域，无需跨域配置
- 开发环境仍可指定独立的后端地址
