# 开发路线图

> 版本：v1.0 | 更新日期：2026-05-25

---

## 总览

项目采用**渐进式分期交付**策略，每期产出可验证的阶段性成果，降低风险。

| 阶段 | 名称 | 周期（估） | 核心产出 |
|------|------|-----------|---------|
| Phase 0 | 项目初始化 | 第1周 | 工程搭建、文档体系 |
| Phase 1 | 基础框架 | 第2-3周 | 导航骨架、主题、路由 |
| Phase 2 | 账号系统 | 第3-4周 | 注册/登录/资料 |
| Phase 3 | 身材系统 | 第5-6周 | 模板选择、拍照录入 |
| Phase 4 | 衣柜管理 | 第7-9周 | 拍照录入、AI抠图、分类 |
| Phase 5 | 虚拟试衣 | 第10-13周 | 拖拽画布、衣物合成、层级 |
| Phase 6 | AI推荐 | 第14-16周 | 搭配生成、保存分享 |
| Phase 7 | 打磨上线 | 第17-18周 | 性能优化、测试、打包 |

---

## Phase 0：项目初始化（当前阶段）

**目标**：搭建开发环境与文档体系

### 任务清单

- [ ] 确认 Flutter 版本并初始化项目
- [ ] 创建项目目录结构（按技术规格书中的结构）
- [ ] 配置 pubspec.yaml 基础依赖
- [ ] 初始化 Git 仓库
- [ ] 配置 CI/CD 基础（GitHub Actions）
- [ ] 创建 Supabase/Firebase 项目
- [ ] 完成 PRD 终稿确认
- [ ] 完成设计规范终稿确认
- [ ] 完成技术规格书终稿确认
- [ ] 输出核心页面线框图

### 验证标准
- `flutter run` 在 Android 和 iOS 模拟器上均能启动
- 基础"Hello World"页面显示正常

## Phase 1：基础框架

**目标**：App 导航壳 + 主题体系 + 空白占位页

### 任务清单

- [ ] 实现主题系统（颜色、字体、间距）
- [ ] 实现底部Tab导航（4个Tab）
- [ ] 创建4个占位页面
- [ ] 封装基础组件（按钮、卡片、输入框、加载态）

### 任务清单

- [ ] 集成 Supabase Auth
- [ ] 实现邮箱注册 + 密码登录
- [ ] 实现第三方登录（Apple ID / Google）
- [ ] 实现密码找回

## Phase 3：身材系统

**目标**：用户可创建/选择虚拟身材模板

### 任务清单

- [ ] 制作5种体型2D插画模板（苹果/H/梨/沙漏/倒三角）
- [ ] 每种体型提供3种肤色
- [ ] 实现模板选择页面（网格/横向滑动）
- [ ] 实现拍照上传身材功能
- [ ] 实现身体轮廓提取（2D剪影）
- [ ] 保存用户选择的身材

### 验证标准
- 选择模板 → 确认 → 进入试衣画布显示模板
- 拍照 → 生成剪影 → 确认 → 进入试衣画布

### 任务清单

- [ ] 实现拍照录入衣物功能
- [ ] 集成AI抠图API（Remove.bg或RMBG-2.0）
- [ ] 实现电商链接导入（输入链接 → 下载图片 → 抠图）
- [ ] 实现衣物分类管理（CRUD）
- [ ] 实现标签系统（颜色、风格、季节）
- [ ] 实现衣物列表页（网格 + 分类Tab）
- [ ] 实现本地存储 + 云端同步

### 验证标准

- 云端同步：A设备录入，B设备可见

## Phase 5：虚拟试衣

**目标**：核心交互——拖拽衣物到身材模板上

### 任务清单

- [ ] 实现试衣画布（Canvas渲染层）
- [ ] 实现身体区域蒙版（基于5种体型分别定义）
- [ ] 实现拖拽交互（从衣柜拖入画布）
- [ ] 实现衣物定位与贴附
- [ ] 实现手动调整（移动、缩放、旋转）
- [ ] 实现多件衣物层级叠加
- [ ] 实现蒙版裁剪（超出身体区域的衣物隐藏）
- [ ] 实现撤销/重置功能
- [ ] 实现搭配结果截图保存
- [ ] 实现分享功能

### 验证标准
- 从衣柜拖一件上衣 → 自动吸附到身体躯干区 → 缩放旋转 → 效果合理
- 上衣+下装+配饰叠加 → 层级正确 → 蒙版裁剪正常

## 风险提示

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| AI抠图效果不理想 | Phase 4卡住 | Phase 4前先验证抠图API效果，备选人工抠图服务 |
| 2D试衣合成效果差 | Phase 5核心体验受损 | Phase 3完成后先做技术验证（POC），确认方案可行再往下走 |
| AI推荐不够智能 | Phase 6体验差 | v1.0先用规则引擎（颜色匹配+风格标签），v2.0再接入大模型 |
| 时间超出预期 | 项目延期 | 每Phase独立可验收，核心功能优先，非核心可裁剪 |
