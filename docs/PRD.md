# 产品需求文档（PRD）— 穿搭辅助App

> 版本：v1.0 | 更新日期：2026-05-25 | 状态：需求确认阶段

---

## 1. 产品定位

一款面向 **18-35岁年轻用户** 的 **2D平面风格穿搭辅助App**，核心解决三大痛点：

| 痛点 | 解决方案 |
|------|---------|
| 出门前反复试穿多套衣服，耗时耗力 | 虚拟试衣画布，拖拽即可预览搭配 |
| 网购无法看上身效果，容易踩雷 | 上传商品图到虚拟模特上预览 |
| 搭配灵感匮乏，不会利用现有衣物 | AI智能推荐搭配方案 |

**平台**：Android + iOS（Flutter 跨平台）  
**商业模式**：当前阶段免费  
**风格**：扁平化插画，清爽干净，无3D效果

---

## 2. 用户角色

| 角色 | 描述 |
|------|------|
| 普通用户 | 注册登录后使用全部功能 |
| 游客 | 不登录可浏览模板，但无法保存衣柜数据 |

---

## 3. 功能模块

### 3.1 账号系统

- 手机号/邮箱注册与登录
- 密码找回
- 第三方登录（微信/Apple ID）
- 个人资料管理（昵称、头像）

### 3.2 「虚拟衣架子」身材系统

**模式A — 预设模板**：
- 提供多种体型的2D扁平插画模板：苹果型、H型、梨形、沙漏型、倒三角型
- 用户直接选择匹配自身的模板即可使用
- 模板为**纯2D平面插画**，无光影、无透视变形、无3D效果

**模式B — 拍照录入**：
- 用户拍摄/上传自己的正面全身照
- 系统提取身体轮廓，生成2D身材剪影

### 3.3 「虚拟衣柜」衣物管理

**录入方式**：
- 拍照上传：对实体衣物拍照，AI自动抠图去背景
- 电商导入：粘贴商品链接或上传商品图片

**分类管理**：
- 预设分类：上衣、下装、连衣裙、外套、配饰、鞋、包
- 标签系统：颜色、风格（休闲/通勤/运动/约会等）、季节（春/夏/秋/冬）
- 来源标记：自有衣物 / 心愿单（未购买）

**存储**：本地为主 + 云端同步

### 3.4 「拖拽式虚拟试衣」画布

- 从衣柜抽屉拖拽衣物到虚拟模特上
- 系统将衣物贴附到对应身体区域
- 支持多件单品叠加：内衣层 → 上衣层 → 外套层 → 配饰层
- 操作手势：
  - 拖拽：移动衣物位置
  - 双指缩放：调整衣物大小
  - 双指旋转：调整衣物角度
- 衣物超出身体轮廓部分按模板蒙版裁剪
- 用户可手动调整层级顺序

### 3.5 「AI智能搭配推荐」

**两个模式**：

| 模式 | 触发方式 | 推荐逻辑 |
|------|---------|---------|
| 全衣柜搭配 | 用户点击"为我搭配" | 基于衣柜中所有衣物，生成3-5套完整搭配 |
| 单品搭配 | 用户选中某件单品后点击"搭配这个" | 推荐能与该单品搭配的其他衣物 |

**推荐范围**：
- 仅推荐自有衣物（默认）
- 推荐电商平台适配商品（需对接电商API）

**搭配结果操作**：
- 一键套用到虚拟试衣画布
- 保存到"我的搭配"收藏夹
- 分享（图片/链接）
- 标记：已尝试 / 已购买

## 4. 界面结构

### 4.1 主导航（底部Tab）

| Tab | 名称 | 功能 |
|-----|------|------|
| 1 | 试衣间 | 虚拟试衣画布 + 身材选择 |
| 2 | 衣柜 | 衣物管理、分类浏览、添加衣物 |
| 3 | 灵感 | AI推荐搭配 |
| 4 | 我的 | 个人中心、收藏、设置 |

### 4.2 关键交互原则

- 以拖拽、滑动为主要交互方式
- 每个操作≤3步完成
- 关键操作有撤销/回退
- 加载状态有明确反馈

---

## 5. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 试衣画布操作延迟 < 100ms；图片加载 < 2s |
| 兼容 | iOS 15+、Android 10+ |
| 存储 | 本地 SQLite + 云端 PostgreSQL |
| 安全 | 用户数据加密传输（HTTPS）、密码哈希存储 |
| 隐私 | 身材数据和照片仅用户本人可见 |

---

## 6. 不做什么（v1.0范围外）

- ❌ 3D建模/渲染
- ❌ AR实景试衣
- ❌ 社交功能（评论、关注、社区）
- ❌ 电商下单闭环
- ❌ 付费/订阅
- ❌ 视频试穿/动态效果
